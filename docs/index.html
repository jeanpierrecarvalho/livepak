livepak

Zero-dep Chrome extension dev server. Auto-reload, toolbar pinning, hot reloading. Just needs Node 22+.

GitHub npm
$ npx livepak ./my-extension

What it does

  • Zero dependencies -- uses Node.js 22+ built-ins (fetch, WebSocket, fs.watch)
  • Auto-downloads Chrome for Testing -- stable Chrome blocks --load-extension, this doesn't
  • Hot reload -- WebSocket-based, file changes trigger instant extension reload
  • Smart reload -- JS/JSON changes do full reload, CSS/HTML just refresh the popup/tab
  • Toolbar pinning -- pins extensions automatically via Chrome DevTools Protocol
  • Multi-extension -- watch and load several extensions at once
  • Crash-safe -- lockfile restores your manifest if the process dies
  • MV2 + MV3 -- works on macOS, Linux, Windows

How it works

Injects a small WebSocket client into your extension's background/service worker. Watches for file changes, then signals chrome.runtime.reload() over the socket. Chrome for Testing is downloaded and cached in ~/.cache/livepak/. Extensions get pinned to the toolbar via CDP. If the process crashes, a lockfile makes sure your manifest gets restored on next run.

Usage

# just run it
npx livepak ./my-extension

# install globally
npm install -g livepak
livepak ./my-extension

# multiple extensions + open a page
livepak ./ext-popup ./ext-content --open https://example.com

# specific Chrome version
livepak ./my-extension --chrome-version 146

# watch-only, no Chrome launch
livepak ./my-extension --no-launch

Options

--port <n> WebSocket port (default: 35729)
--open <url> Open a URL after launch
--chrome-version <v> Chrome version ("146", "latest")
--chrome-flags <f> Extra flags, comma-separated
--no-launch Just watch, don't open Chrome
--no-inject Skip reload script injection
--config <path> Config file (default: ./livepak.config.json)

Config file

Optional livepak.config.json in your project root:

{
  "extensions": ["./src/extension"],
  "port": 4000,
  "open": "https://example.com",
  "chromeVersion": "146",
  "chromeFlags": ["--auto-open-devtools-for-tabs"]
}
