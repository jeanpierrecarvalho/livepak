livepak

Chrome extension dev server with auto-reload, toolbar pinning, and hot reloading. No npm install needed.

GitHub npm
$ npx livepak ./my-extension click to copy

Features

Zero dependencies
No npm install. Uses Node.js 22+ built-ins: native fetch, WebSocket, fs.watch.
Auto-downloads Chrome
Downloads Chrome for Testing automatically. Bypasses stable Chrome's --load-extension block.
Hot reload
File changes trigger instant extension reload via WebSocket. No manual refresh.
Smart reload
JS/JSON = full reload. CSS/HTML = soft reload + tab refresh. Content scripts reload tabs via CDP.
Auto-pins to toolbar
Extensions are pinned to the Chrome toolbar automatically using the DevTools Protocol.
Multi-extension
Load and watch multiple extensions simultaneously. Each gets its own file watcher.
Crash-safe
Lockfile stores original manifests. If the process dies, next startup restores everything.
MV2 + MV3
Works with both Manifest V2 and V3 extensions on macOS, Linux, and Windows.

How it works

Injects a tiny WebSocket client into your extension's service worker or background script

Watches your extension directory for file changes using native fs.watch

Downloads Chrome for Testing and caches it in ~/.cache/livepak/

Launches Chrome with --load-extension and pins extensions via CDP

Signals chrome.runtime.reload() over WebSocket when files change

Refreshes active tabs when content scripts may have changed

Restores your original manifest on exit (or on next startup if crashed)

Usage

Basic 
# run directly with npx
npx livepak ./my-extension

# or install globally
npm install -g livepak
livepak ./my-extension
Advanced 
# multiple extensions
livepak ./ext-popup ./ext-content-script

# test content scripts on a specific page
livepak ./my-extension --open https://example.com

# use a specific Chrome version
livepak ./my-extension --chrome-version 146

# watch-only mode (bring your own browser)
livepak ./my-extension --no-launch

CLI options

--port <n> WebSocket port (default: 35729, auto-increments on conflict)
--open <url> Open a URL after launch (for testing content scripts)
--chrome-version <v> Chrome version (e.g. "146", "latest")
--chrome-flags <flags> Extra Chrome flags, comma-separated
--no-launch Don't launch Chrome, just watch and reload
--no-inject Don't inject reload script (manual setup)
--config <path> Config file path (default: ./livepak.config.json)

Config file

Create livepak.config.json in your project root:

livepak.config.json 
{
  "extensions": ["./src/extension"],
  "port": 4000,
  "open": "https://example.com",
  "chromeVersion": "146",
  "chromeFlags": ["--auto-open-devtools-for-tabs"]
}
