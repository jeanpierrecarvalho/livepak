livepak

Dev server for Chrome extensions. Hot reload, auto-pinning, zero dependencies.

GitHub npm
$ npx livepak ./my-extension

Why

Developing Chrome extensions means constantly reloading: remove, load unpacked, refresh, test. Stable Chrome doesn't even allow --load-extension. livepak handles all of that -- downloads Chrome for Testing, watches your files, reloads the extension over WebSocket, and pins it to the toolbar. No dependencies, just Node 22+.

Features

  • No dependencies -- Node 22+ has everything (fetch, WebSocket, fs.watch)
  • Downloads Chrome for Testing -- cached in ~/.cache/livepak/
  • Hot reload -- file changes trigger chrome.runtime.reload()
  • Smart reload -- JS/JSON = full reload, CSS/HTML = soft refresh
  • Auto-pins extensions to the toolbar via CDP
  • Multi-extension -- load and watch several at once
  • Crash recovery -- lockfile restores manifests if the process dies
  • Manifest V2 and V3, macOS/Linux/Windows

Usage

# just run it
npx livepak ./my-extension

# or install globally
npm i -g livepak
livepak ./my-extension

# multiple extensions, open a test page
livepak ./popup ./content --open https://example.com

# pin to a Chrome version
livepak ./my-extension --chrome-version 146

# watch only, bring your own browser
livepak ./my-extension --no-launch

Options

--port <n> WebSocket port (default 35729)

--open <url> open a URL after launch

--chrome-version <v> e.g. "146", "latest"

--chrome-flags <f> extra flags, comma-separated

--no-launch skip Chrome, just watch + reload

--no-inject don't inject the reload script

--config <path> config file path

Config

Drop a livepak.config.json in your project root:

{
  "extensions": ["./src/extension"],
  "port": 4000,
  "open": "https://example.com",
  "chromeVersion": "146",
  "chromeFlags": ["--auto-open-devtools-for-tabs"]
}
